# Локальный запуск для ручного тестирования (Windows 11)

Этот вариант запускает проект **без Docker и без MySQL** — на встроенной базе
SQLite и локальном PHP-сервере. Подходит для быстрого ручного тестирования.

## Требования

- **PHP 8.2+** в `PATH` (проверка: `php -v`)
- **Node.js 18+** и **npm** в `PATH` (проверка: `node -v`, `npm -v`)
- Composer устанавливать необязательно — скрипт скачает локальный `composer.phar`.

## Установка (один раз)

Дважды кликните по **`setup.bat`** (или запустите в терминале):

```bat
setup.bat
```

Скрипт:
1. скачает `composer.phar` (если нужно);
2. установит зависимости PHP и npm;
3. создаст `.env` и ключ приложения;
4. создаст базу SQLite и наполнит её тестовыми данными;
5. соберёт фронтенд.

> Если `.env` уже настроен под SQLite (как в этом репозитории после подготовки),
> `setup.bat` его не перезапишет.

## Запуск

Дважды кликните по **`start.bat`**. Откроются три окна (веб-сервер, Vite, обработчик
очередей) и браузер на <http://127.0.0.1:8000>.

Остановка — закрыть открывшиеся окна консоли.

## Полезные команды

```bat
REM Пересоздать базу с нуля и заново наполнить данными
php artisan migrate:fresh --seed --force

REM Только веб-сервер
php artisan serve --host=127.0.0.1 --port=8000

REM Фронтенд с hot-reload
npm run dev

REM Просмотр писем: при MAIL_MAILER=log письма пишутся в storage/logs/laravel.log
```

## Примечания

- Почта настроена на драйвер `log` — реальные письма не отправляются, их содержимое
  попадает в `storage/logs/laravel.log`. Это удобно для проверки писем активации и
  сброса пароля без SMTP-сервера.
- Очереди используют базу данных; окно «Queue worker» обрабатывает уведомления.
- Telegram-бот и Google OAuth по умолчанию отключены (пустые токены в `.env`).
